# Antigravity CLI 기능

### 플러그인 

**플러그인 작동 방식**  
플러그인은 스킬(skills), 에이전트(agents), 규칙(rules), MCP 서버, 훅(hooks)을 단일 배포 가능 단위로 포함할 수 있는 네임스페이스 번들입니다.

플러그인을 설치하면 CLI는 해당 파일들을 홈 디렉터리의 `~/.gemini/antigravity-cli/plugins/<plugin_name>/` 아래에 스테이징(staging)합니다. Antigravity 에이전트는 이 스테이징된 맞춤 설정을 자동으로 감지하고 로드합니다.

```
~/.gemini/antigravity-cli/
├── plugins/
│   └── <plugin_name>/
│       ├── plugin.json         # 필수 마커 파일
│       ├── mcp_config.json     # 선택적 MCP 서버 정의
│       ├── hooks.json          # 선택적 이벤트 훅 정의
│       ├── skills/             # 선택적 스킬
│       ├── agents/             # 선택적 서브에이전트
│       └── rules/              # 선택적 규칙
└── import_manifest.json        # 추적 매니페스트
```

**플러그인 구성 요소 접근**  
스테이징되고 로드되면 슬래시 명령어를 사용하여 CLI 내에서 플러그인 구성 요소와 상호 작용할 수 있습니다.

### 터미널 샌드박스 

터미널 샌드박스는 에이전트가 로컬 셸 명령어를 실행할 때 호스트 시스템을 잠재적으로 파괴적인 파일 조작이나 권한 없는 외부 네트워크 요청으로부터 보호하는 경량 보안 격리 메커니즘입니다.

무거운 가상 머신이나 컨테이너를 실행하는 대신, CLI는 운영 체제 고유 기능(Linux의 `nsjail`, macOS의 `sandbox-exec`, Windows의 `AppContainer`)을 활용하여 시작 오버헤드 없이 엄격한 격리 경계를 적용합니다.

**설정**  
`settings.json` 파일(`~/.gemini/antigravity-cli/settings.json`에 위치)에서 샌드박스 동작을 설정할 수 있습니다.

```json
{
  "enableTerminalSandbox": true
}
```

* **`enableTerminalSandbox`** (boolean, 기본값: `false`): 모든 로컬 에이전트 프로세스에 일반적인 실행 격리 장벽을 활성화합니다.

**대화형 승인**  
에이전트가 사용자 확인이 필요한 터미널 명령어를 제안할 때 CLI 프롬프트는 설정에 따라 동적으로 변경됩니다.

* **샌드박스가 활성화된 경우**: 신뢰할 수 있는 단일 명령어에 대해 임시로 격리 경계를 우회해야 하는 경우를 위해, 확인 프롬프트에 **예, 샌드박스 제한 없이 실행**하는 특정 옵션이 포함됩니다.  
* **샌드박스가 비활성화된 경우**: 잠재적으로 위험할 수 있는 특정 명령어를 안전 경계 내에서 강제로 실행하고 싶을 때를 위해, 프롬프트에 **예, 샌드박스 내에서 실행**하는 옵션이 포함됩니다.

### CLI 슬래시 명령어 참조

Antigravity CLI는 대화를 관리하고, 설정을 구성하며, 에이전트 기능을 검사하기 위해 프롬프트 입력창에 직접 입력할 수 있는 다양한 슬래시 명령어를 지원합니다.

### 핵심 슬래시 명령어

| 명령어 | 카테고리 | 목적 |  
| :--- | :--- | :--- |  
| **`/resume`** *(별칭 `/switch`)* | 대화 | 세션을 재개하거나 전환하기 위해 대화 선택기를 엽니다. |  
| **`/rewind`** *(별칭 `/undo`)* | 대화 | 대화 기록을 이전 체크포인트로 되돌립니다. |  
| **`/rename <name>`** | 대화 | 더 쉽게 추적할 수 있도록 활성화된 대화 스레드의 이름을 변경합니다. |  
| **`/permissions`** | 설정 구성 | 에이전트 자율성 수준(`request-review`, `always-proceed` 또는 `strict`)을 선택합니다. |  
| **`/model`** | 설정 구성 | 기본 추론 모델을 선택합니다 (세션 간에 유지됨). |  
| **`/keybindings`** | 설정 구성 | 대화형 키보드 단축키 에디터를 엽니다. |  
| **`/statusline`** | 설정 구성 | CLI 상태 표시줄에 나타나는 실시간 지표를 커스텀 설정합니다. |  
| **`/tasks`** | 도구 및 모니터링 | 활성 백그라운드 태스크를 모니터링하고, 로그를 보거나 종료합니다. |  
| **`/skills`** | 도구 및 모니터링 | 로컬 및 글로벌에 캡슐화된 에이전트 워크플로를 탐색합니다. |  
| **`/mcp`** | 도구 및 모니터링 | Model Context Protocol 서버를 구성하고 관리하는 패널을 엽니다. |  
| **`/open <path>`** | 유틸리티 | 기본 설정된 외부 에디터로 파일을 즉시 엽니다. |  
| **`/usage`** | 유틸리티 | 터미널 내부에서 인라인 인터랙티브 도움말 설명서를 엽니다. |  
| **`/logout`** | 계정 | Google 세션에서 로그아웃하고 캐시된 자격 증명을 지웁니다. |

### `settings.json`을 통한 고급 커스텀 설정

고급 사용자의 경우, 몇 가지 슬래시 명령어는 `~/.gemini/antigravity-cli/settings.json` 설정을 통해 세부적인 커스텀을 지원합니다.

* **세부 권한 관리**: 전역 수준 대신 허용/거부할 특정 명령어를 정의합니다.  
  ```json  
  "permissions": {  
    "allow": ["command(git)", "command(npm test)"],  
    "deny": ["command(rm -rf)"]  
  }  
  ```  
* **사용자 정의 상태 표시줄 및 창 제목**: 실시간 에이전트 메타데이터(CWD, 활성 모델, 토큰 사용량, 상태 등이 포함된 JSON 형식)를 커스텀 셸 스크립트로 직접 전달하여 동적인 상태 표시줄이나 터미널 창 제목을 생성할 수 있습니다.

### Antigravity CLI의 서브에이전트 

Antigravity CLI는 메인 에이전트가 대화를 중단하지 않고도 병렬 작업을 위임하고 백그라운드 조사를 수행하며 시스템 테스트를 실행할 수 있게 해주는 비동기 서브에이전트 프레임워크를 제공합니다.

**서브에이전트란 무엇인가요?**  
서브에이전트는 메인 대화와 병렬로 특정 백그라운드 태스크를 처리하기 위해 설계된 독립적이고 동시 실행되는 에이전트 세션입니다.

* **목적**: 메인 에이전트는 문서 검색, 빌드 실행, 수정 사항 검증 등과 같은 백그라운드 작업을 수행하기 위해 서브에이전트를 자동으로 생성합니다.  
* **기능**: 서브에이전트는 할당된 태스크를 완료하기 위해 코드 검색, 파일 편집, 터미널 명령어, 웹 검색 등의 도구에 완전히 접근할 수 있습니다.  
  * 메인 에이전트는 서브에이전트에게 부여할 도구와 권한을 결정하며, 여기에는 MCP 도구 사용 여부 및 파일 작성 허용 여부가 포함됩니다.

### 에이전트 관리: `/agents` 패널

Antigravity CLI는 실행 중인 서브에이전트를 확인하고 관리하며 해당 조치를 승인할 수 있는 대화형 터미널 UI를 제공합니다.

* **Access**: 프롬프트 창에 `/agents`를 입력하면 서브에이전트 패널이 열립니다.  
* **개요**: 패널에는 활성 상태 및 완료된 서브에이전트 목록이 표시되며, 상태(running, done, killed 등) 및 현재 실행 중인 단계와 같은 개략적인 세부 정보가 포함됩니다.

**도구 확인 및 승인**  
서브에이전트가 사용자 권한이 필요한 도구(예: 로컬 명령어 실행 또는 파일 작성)를 실행하려는 경우, 요청을 제안합니다. 다음과 같은 두 가지 방법으로 승인을 관리할 수 있습니다. 

1. **상세 뷰 승인**  
   서브에이전트 상세 뷰에는 모든 대기 중인 승인 요청이 포함된 상호작용 섹션이 있으며, 여기서 승인 또는 거부를 선택적으로 할 수 있습니다.  
     
   <Announcement>
   icon: lightbulb
   iconColor: var(--theme-primary)
   color: var(--theme-surface-surface-container)
   text: **팁**: 키보드 단축키 `ctrl+j`를 누르면 메인 대화에서 승인 대기 중인 다음 서브에이전트의 상세 뷰로 바로 "텔레포트"할 수 있습니다.
   </Announcement>
     
2. **빠른 알림**  
   원활한 작업 흐름을 유지할 수 있도록, 서브에이전트가 권한을 요청할 때 Antigravity CLI는 프롬프트 입력창 바로 위에 빠른 승인 알림(Fast Path Alert)을 표시합니다.  
     
   <Announcement>
   icon: lightbulb
   iconColor: var(--theme-primary)
   color: var(--theme-surface-surface-container)
   text: **팁**: 메인 대화에서 다른 화면으로 전환하지 않고도 `ctrl+k`를 사용하여 대기 중인 서브에이전트 권한 요청을 즉시 승인할 수 있습니다.
   </Announcement>
