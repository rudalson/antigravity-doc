# Antigravity CLI 개요

Antigravity CLI는 Antigravity의 가벼운 터미널 사용자 인터페이스(TUI) 환경입니다. 다단계 추론, 다중 파일 편집, 도구 호출, 대화 기록 등 Antigravity 2.0의 동일한 핵심 에이전트 기능을 터미널에 직접 제공합니다.

## 왜 Antigravity CLI인가요?

Antigravity CLI는 공유 에이전트 하네스의 추론, 실행 및 오케스트레이션 기능을 로컬 쉘로 직접 가져옵니다. Antigravity 2.0은 포괄적인 시각적 에디터 인터페이스를 제공하는 반면, CLI는 빠른 속도, 가벼운 리소스 소모 및 터미널 중심 워크플로우와의 원활한 통합을 위해 특화 설계되었습니다.

### 플랫폼 비교

| 기능 | Antigravity CLI | Antigravity 2.0 |
| :--- | :--- | :--- |
| **기본 인터페이스** | 키보드 중심의 TUI | 시각적 데스크톱 에디터 / IDE |
| **성능 오버헤드** | 거의 없음, 극도로 가벼움 | 표준 데스크톱 IDE 수준의 리소스 소모 |
| **워크플로우 포커스** | 빠른 로컬 반복 작업, SSH, 헤드리스(headless) | 전체 프로젝트 관리, 시각적 작업 공간 |
| **내비게이션** | 유니버설 키보드 단축키 | 마우스 및 다중 패널 레이아웃 |
| **원격 사용성** | 네이티브 SSH, tmux 및 터미널 멀티플렉서 지원 | 로컬 작업 공간 또는 원격 개발 컨테이너 |

## 통합 기능

Antigravity CLI는 Antigravity 2.0과 연동하여 작동하며, 설정을 공유하고 인터페이스 간의 마찰 없는 전환을 가능하게 합니다.

- **공유 에이전트 하네스**: 두 환경 모두 정확히 동일한 에이전트 코어에서 실행됩니다. 다단계 추론, 도구 사용 또는 코드 이해도와 관련된 모든 개선 사항이 두 플랫폼 모두에 동일하게 적용됩니다.
- **공유 설정 동기화**: 핵심 기본 설정, 권한 및 보안 구성이 두 인터페이스 간에 자동으로 동기화됩니다. 한 플랫폼에서 권한 규칙이나 표준 설정을 업데이트하면 다른 플랫폼에도 즉시 반영됩니다.
- **대화 내보내기**: 플랫폼 간에 진행 중인 대화를 원활하게 이동할 수 있습니다. 터미널 세션의 복잡도가 증가하여 시각적 조율이 필요한 경우, 대화를 Antigravity 2.0으로 내보내 시각적 에디터 인터페이스에서 계속 진행할 수 있습니다.

## Gemini CLI에서 마이그레이션하기

Gemini CLI에서 전환하는 경우, 기존 Gemini CLI 확장 기능, 스킬 및 설정을 자동으로 마이그레이션하는 일회성 가져오기 프로세스가 지원됩니다. 자세한 내용은 [Gemini CLI에서 마이그레이션하기](/docs/gcli-migration)를 참조하세요.

## 다음 단계

아래 가이드를 살펴보고 환경을 설정한 뒤 자율 에이전트와 함께 작업을 시작해 보세요.

- **[설치 및 인증](/docs/cli-install)**: CLI를 설치하고, 기업용 매개변수를 구성하며, 사일런트(silent) 인증을 완료합니다.
- **[시작하기](/docs/cli-getting-started)**: 온보딩 로드맵, 최초 실행 설정 및 핵심 개념 모델을 살펴봅니다.
- **[튜토리얼](/docs/cli-tutorial)**: 활성 에이전트와 함께 첫 번째 다중 파일 생성 작업을 실행해 봅니다.
- **[프롬프트 및 상호작용](/docs/cli-prompting)**: 다중 라인 작성, 프롬프트 편집 및 터미널 미디어 붙여넣기 기능을 마스터합니다.
- **[아티팩트 검토](/docs/cli-artifacts)**: 투명성을 활용하여 에이전트의 계획, diff 및 테스트 실행을 검토합니다.
- **[AI 크레딧](/docs/cli-credits)**: AI Premium 크레딧 폴백, 가격 링크 및 설정을 구성하고 모니터링합니다.
- **[플러그인 및 스킬](/docs/cli-plugins)**: 자신만의 커스텀 스킬 슬래시 명령을 만들고, 훅을 관리하며, MCP 서버를 구성합니다.
- **[모범 사례](/docs/cli-best-practices)**: 워크플로우 파이프라인, 검증 루프 및 세션 진행 방향 수정(course-correction)을 마스터합니다.
