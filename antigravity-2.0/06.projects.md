# 프로젝트

Antigravity 2.0에서는 기존의 리포지토리 중심 워크스페이스 모델에서 보다 유연하고 안전한 **프로젝트 중심** 모델로 전환하고 있습니다. 이 문서는 프로젝트가 무엇인지, 어떻게 작동하는지, 그리고 기존 워크스페이스 구조와 어떻게 다른지 설명합니다.

### 프로젝트(Project)란 무엇인가요?

**프로젝트(Project)**는 에이전트의 환경과 범위를 정의하는 폴더 구성입니다. 에이전트가 단일 폴더 내에서만 동작하도록 강제하는 대신, 프로젝트는 프론트엔드 및 백엔드 리포지토리와 같이 하나 또는 여러 폴더를 대상으로 작동할 수 있어 코드베이스에 필요한 모든 컨텍스트를 에이전트에게 제공할 수 있습니다. 모든 프로젝트는 격리된 고유한 에이전트 설정을 가지므로 서로 다른 프로젝트의 보안 설정을 개별적으로 맞춤 구성할 수 있습니다.

### 주요 차이점: 워크스페이스 vs. 프로젝트

| 기능 | 기존 모델 (워크스페이스) | 새로운 모델 (프로젝트) |  
| :--- | :--- | :--- |  
| **조직 범위** | 단일 로컬 리포지토리와 긴밀하게 결합되어 있습니다. | 프로젝트는 에이전트가 작업해야 하는 모든 컨텍스트와 폴더의 구성입니다. |  
| **디렉터리 경계** | 에이전트가 하나의 폴더 구조 내로 엄격하게 한정됩니다. | 단일 프로젝트가 한 번에 **여러 폴더**에 걸쳐 있을 수 있습니다. |  
| **설정 격리** | 머신에서 전역적으로 설정을 상속받습니다. | 프로젝트는 고유한 설정을 가집니다. 프로젝트 내의 에이전트는 해당 프로젝트의 설정을 사용합니다. |  
| **권한** | 넓은 범위의 전역 권한입니다. | 전역 권한을 상속받습니다. 프로젝트는 전역 권한 외에도 고유한 자체 권한을 가질 수 있습니다. |  
| **사용자 정의** | 스킬/MCP가 전역 또는 워크스페이스별로 관리됩니다. | 재사용 가능한 스킬, MCP 및 훅을 전역적으로 상속받습니다. |

### 핵심 프로젝트 개념

**1. 폴더 (Folders)**  
프로젝트는 에이전트가 액세스할 수 있는 디렉터리와 리포지토리를 정의하는 **폴더**로 구성됩니다:  
* **로컬 폴더**: Git이 설정되지 않은 폴더입니다.  
* **로컬 Git 체크아웃**: Git 리포지토리 체크아웃 폴더입니다.

**2. 워크트리 선택 (로컬 vs. 새 워크트리)**  
프로젝트에서 새 대화를 시작할 때, 워크트리 선택기를 통해 에이전트가 폴더와 상호작용하는 방식을 선택합니다:

* **Local Mode**(로컬 모드): 에이전트가 활성 로컬 폴더 또는 Git 체크아웃에서 직접 작업합니다. (현재 작업 폴더에서 신속하고 대화식인 편집을 수행하는 데 적합합니다).  
* **New Worktree Mode**(새 워크트리 모드): 대화를 위해 새 Git 워크트리를 생성합니다. (복잡한 작업에 적합하며, 현재 활성화된 작업 폴더를 그대로 유지하고 병렬 서브에이전트 간의 충돌을 방지합니다).

**3. 범위가 지정된 설정 및 권한**  
설정과 권한은 모두 프로젝트 수준에서 범위가 지정됩니다:  
* **설정**: 프로젝트가 생성되면 항상 기본 보안 사전 설정으로 시작되며, 프로젝트의 모든 폴더에 대한 읽기 및 쓰기 권한을 가지고 모든 터미널 명령 실행 시 승인을 요청합니다. 이 설정은 변경이 가능하며 해당 프로젝트 내의 모든 에이전트에 적용됩니다.  
* **권한**: 프로젝트는 전역 권한을 상속받지만 프로젝트 수준에서 권한을 강화할 수 있으므로, 에이전트가 특정 프로젝트 작업에 정확히 필요한 만큼의 권한만 가지도록 보장합니다.

### 프로젝트를 활용한 워크플로우

* **단일 폴더에서 작업하기**: 폴더 하나로 프로젝트를 생성한 후 설정을 구성합니다.  
* **여러 폴더에서 작업하기**: 에이전트가 여러 코드베이스를 아우르는 전체 맥락을 가질 수 있도록 관련된 모든 폴더를 단일 프로젝트에 추가합니다.  
* **동일한 폴더에서 병렬 에이전트 실행하기**: 모든 에이전트가 동일한 활성 폴더에서 작업하도록 에이전트를 시작할 때 **Local Mode**(로컬 모드)를 선택합니다.  
* **동시 에이전트 격리하기**: 각 에이전트 세션별로 별도의 격리된 Git 워크트리가 임시 프로비저닝되도록 에이전트를 시작할 때 **New Worktree Mode**(새 워크트리 모드)를 선택하여 에이전트 간의 충돌을 방지합니다.  
* **체크아웃 및 로컬 폴더 혼합 사용**: 로컬 모드에서의 작업은 기존 폴더에서 직접 작동합니다. \"New Worktree Mode\"를 사용하면 활성화된 모든 Git 체크아웃에 대해 새 Git 워크트리가 가동되어, 에이전트가 새 워크트리 내부와 기존의 Git 미설정 로컬 폴더에서 동시에 작동할 수 있게 됩니다.
