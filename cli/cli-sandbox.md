# 샌드박스

운영 체제 고유의 프로세스 격리를 강제 적용하고, 실행 격리 경계를 관리하며, 로컬 워크스테이션을 보호하세요.

## 보안 모델 

자율 개발 에이전트는 로컬 터미널 명령어를 실행하고, 소스 코드를 편집하며, 작업 공간에서 직접 테스트를 수행하기 때문에 안전한 워크스테이션 환경을 유지하는 것이 매우 중요합니다. Antigravity CLI는 파괴적인 셸 작업이나 권한이 없는 원격 네트워크 호출을 제한하기 위해 네이티브 **터미널 샌드박스(Terminal Sandbox)**를 탑재하고 있습니다.

### 운영 체제 고유의 격리 

실행 속도를 지연시키는 무거운 가상 컨테이너나 격리된 가상 머신과 달리, Antigravity는 가볍고 네이티브한 운영 체제 커널 유틸리티를 사용하여 실행 오버헤드 없이 안전한 프로세스 링을 생성합니다.

| 운영 체제 | 샌드박스 유틸리티 | 보안 특징 |
| :---------------------------- | :----------------- | :---------------------------------------------------------------------------------------------------------------- |
| **Linux**                     | `nsjail`           | CPU, 메모리 및 경로 가시성을 격리하기 위해 커널 네임스페이스와 cgroups를 사용하는 오픈소스 프로세스 격리 툴입니다. |
| **macOS**                     | `sandbox-exec`     | 절대 경로 파일 시스템 액세스 및 원시 TCP 쿼리를 제한하는 정책 프로필을 적용하는 네이티브 시스템 툴입니다. |
| **Windows**                   | `AppContainer`     | 파일 시스템 권한 및 레지스트리 가시성을 격리하는 데스크톱 보안 격리 링입니다. |

## 샌드박스 활성화

글로벌 설정 내에서 직접 샌드박스를 구성할 수 있습니다.

```text
~/.gemini/antigravity-cli/settings.json
```

### 샌드박스 설정

설정 프로필에 샌드박스 활성화 토글을 추가합니다.

```json
{
  "enableTerminalSandbox": true
}
```

- **`enableTerminalSandbox`** (boolean, 기본값: `false`): 에이전트가 실행하는 모든 로컬 실행 명령어를 OS 격리 링으로 제한합니다.

## 샌드박스를 적용한 대화형 승인

에이전트가 터미널 도구 또는 셸 명령어를 실행하려고 시도할 때, TUI 프롬프트 창은 샌드박스 상태에 따라 동적으로 변경됩니다.

- **샌드박스가 활성화된 경우**: 프롬프트 패널에서 일시적으로 샌드박스를 우회할 수 있는 옵션을 제공합니다.
  ```text
  진행하시겠습니까?
  1. 예
  2. 예, 샌드박스 제한 없이 실행
  3. 아니요
  ```
  옵션 2를 선택하면 해당 단일 실행에 한해 격리 장벽을 우회합니다.
- **샌드박스가 비활성화된 경우**: 위험한 명령어에 대해 강제로 격리를 적용할 수 있는 옵션을 제공합니다.
  ```text
  진행하시겠습니까?
  1. 예
  2. 예, 샌드박스 내에서 실행
  3. 아니요
  ```
