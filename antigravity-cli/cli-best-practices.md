# Antigravity CLI 모범 사례

워크플로, 프롬프트 아키텍처 및 로컬 구성 선택을 마스터하여 강력한 제어력을 유지하면서 에이전트 속도를 극대화하십시오.

## 검증 루프 구축

자율 에이전트로부터 신뢰할 수 있고 올바른 수정을 보장하는 가장 효과적인 방법은 에이전트에 로컬 검증 메커니즘(예: 단위 테스트, 빌드 명령 또는 형식 지정 스크립트)을 제공하는 것입니다.

에이전트에게 코드 변경 구현을 요청하기 전에 다음을 수행합니다.

1. 워크스페이스 디렉터리에 테스트 모음이 준비되어 있는지 확인합니다.
2. 테스트가 존재하지 않는 경우, 에이전트에게 표준 테스트 블록을 _먼저_ 작성하도록 지시합니다.
3. 에이전트가 코드를 제안하면 로컬 테스트 명령을 실행하여 작업을 검증하도록 지시합니다.
4. 에이전트가 명령을 실행하고 테스트 출력을 자동으로 반복하는 것을 확인합니다.

```text
> main.py에 X 기능을 구현하세요. 그 후 npm test를 실행하여 빌드를 검증하세요.
```

## 탐색, 계획, 실행

자율 로컬 에이전트는 복잡한 변경 사항이 고유한 탐색, 계획 및 실행 단계로 분할될 때 가장 높은 정확도로 작동합니다.

- **탐색(Exploration)**: 변경 사항을 작성하기 전에 대상 코드베이스가 특정 문제를 어떻게 해결하는지 또는 인터페이스가 정의된 위치를 설명하도록 에이전트에 요청합니다.
- **계획(Planning)**: 구현 계획을 요청합니다. 에이전트는 구현 계획 아티팩트에 대상 파일, 필수 종속성 및 논리 재정의를 나열합니다.
- **실행(Execution)**: 구조화된 계획을 승인하면 에이전트에게 편집을 적용하도록 지시합니다.

```text
> 우리 라우터가 `/docs/:page`를 어떻게 해결하는지 탐색하세요. `/docs/best-practices`를 추가하기 위한 구현 계획을 작성하세요.
```

## 프롬프팅 컨텍스트 풍부화

로컬 에이전트에게 고정밀 지표를 제공하여 추론 경계를 좁히고 토큰 오버헤드를 최소화합니다.

### 대상 파일 자동 완성

프롬프트 상자 안에 `@`를 입력하여 **대화형 경로 제안** 오버레이를 트리거합니다. 경로를 강조 표시하고 선택하면 절대 워크스페이스 파일 경로가 프롬프트에 직접 가져와집니다. 이는 에이전트가 코드 검색 대상을 지정하는 데 도움이 됩니다.

### 시각적 증거 첨부

시각적 UI 문제, 렌더링 버그 또는 프론트엔드 레이아웃 불일치를 디버깅하는 경우 스크린샷이나 비디오 녹화를 캡처하여 복사한 다음 프롬프트 상자 내부에서 `ctrl+v`를 눌러 첨부합니다. 에이전트는 미디어 파일을 참조하여 문제를 진단합니다.

## 워크스페이스 환경 구성

엔지니어링 흐름에 맞게 로컬 워크스테이션 규칙 및 보안 경계를 최적화합니다.

### 코드베이스 규칙 파일 작성

워크스페이스 루트에 `GEMINI.md` 또는 `AGENTS.md` 파일을 생성하여 구체적인 디렉터리 표준, 스타일 패러다임, 테스트 명령 매개변수 및 지원 중단 경고를 개략적으로 설명합니다. 에이전트는 시작 시 이러한 규칙을 자동으로 분석하고 변경 사항을 제안하기 전에 참조합니다.

### 구조화된 권한 설정

프로젝트 위험 수준에 따라 `~/.gemini/antigravity-cli/settings.json`에서 안전 장벽을 조정합니다.

- **`request-review`** (기본값): 쓰기 작업, bash 명령 또는 원격 네트워크 호출을 실행하기 전에 확인 메시지를 표시합니다.
- **`proceed-in-sandbox`**: 모든 터미널 실행을 안전한 샌드박스 격리 링으로 제한합니다. 안전한 명령은 자율적으로 실행되는 반면 위험한 명령은 검토를 요청합니다.
- **`strict`**: 모든 비읽기 작업에 대해 항상 확인 메시지를 표시하여 완전한 한 줄 한 줄 투명성을 제공합니다.

```json
{
  "toolPermission": "proceed-in-sandbox",
  "enableTerminalSandbox": true
}
```

## 적극적인 TUI 세션 관리

활성 세션 탐색 도구를 사용하여 엔지니어링 막다른 골목에서 복구하거나 중간 에이전트 루프의 경로를 수정합니다.

### 조기 경로 수정 (`esc`)

에이전트가 잘못된 검색 패턴을 실행하거나 의도와 다른 코드를 작성하는 것을 감시하는 경우, 즉시 전역 탈출 키인 `esc`를 눌러 차례를 중단하고 깨끗한 프롬프트의 포커스를 되찾습니다.

### `/rewind`로 기록 되감기

에이전트가 빌드 오류를 발생시키는 여러 연속적인 변경을 수행한 경우 세션을 무시할 필요가 없습니다. 대화 스레드를 이전의 안정적인 체크아웃으로 되돌리려면 `/rewind`(또는 `/undo`)를 입력합니다.

### `/fork`로 실험 브랜치 생성

가장 좋은 구현 경로가 확실하지 않은 경우:

1. 안정적인 기본 스레드에 도달합니다.
2. `/fork`를 입력하여 복제된 병렬 세션을 가동합니다.
3. 분기된 세션에서 투기적 코드 수정을 테스트합니다.
4. 접근 방식이 실패하면 `/resume`을 실행하여 안정적인 메인 브랜치로 다시 교체합니다.

## 자동화 및 스크립팅

Antigravity CLI는 표준 쉘 파이프라인 도구 내에서 원활하게 작동하도록 설계되었습니다.

### 비대화형 명령 실행 (`-p`)

빠른 쿼리를 자동화하거나 에이전트를 git 훅에 통합하려면 원샷 프롬프트 플래그 `-p`를 사용합니다.

```bash
agy -p "Review this git diff and draft a conventional commit message" --cwd $(pwd)
```

### 병렬 서브에이전트를 사용하여 팬아웃

대규모 스윕 또는 다중 파일 리팩터링의 경우, 메인 에이전트에게 동시 백그라운드 서브에이전트를 생성하도록 지시합니다. 에이전트 관리자는 주 화면에서 작업을 계속하는 동안 백그라운드 스레드를 자율적으로 처리합니다.
