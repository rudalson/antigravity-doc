# 규칙 (Rules)

규칙은 에이전트가 로컬 및 전역 수준 모두에서 준수하도록 수동으로 정의된 제약 조건입니다. 규칙을 통해 사용자는 자신의 사용 사례와 스타일에 특화된 동작을 따르도록 에이전트를 안내할 수 있습니다.

규칙을 시작하려면:
1. 에이전트 패널 상단의 "..." 드롭다운을 통해 Customizations(사용자 정의) 패널을 엽니다.
2. Rules(규칙) 패널로 이동합니다.
3. 새로운 전역 규칙을 생성하려면 **+ Global**을 클릭하고, 특정 워크스페이스 전용 규칙을 생성하려면 **+ Workspace**를 클릭합니다.

규칙 자체는 단순한 마크다운 파일이며, 여기에 에이전트가 작업, 기술 스택 및 스타일을 수행하는 데 필요한 제약 조건을 입력할 수 있습니다.

규칙 파일은 각각 최대 12,000자까지로 제한됩니다.

## 전역 규칙 (Global Rules)

전역 규칙은 `~/.gemini/GEMINI.md`에 위치하며 모든 워크스페이스에 걸쳐 적용됩니다.

## 워크스페이스 규칙 (Workspace Rules)

워크스페이스 규칙은 워크스페이스 또는 git 루트의 `.agents/rules` 폴더에 위치합니다.

규칙 수준에서 규칙이 활성화되는 방식을 정의할 수 있습니다:

- 수동(Manual): 에이전트의 입력창에서 @ 멘션을 통해 규칙이 수동으로 활성화됩니다.
- 항상 켜짐(Always On): 규칙이 항상 적용됩니다.
- 모델 판단(Model Decision): 규칙에 대한 자연어 설명을 기반으로 모델이 규칙 적용 여부를 결정합니다.
- Glob: 정의한 glob 패턴(예: `*.js`, `src/**/*.ts`)을 기반으로 해당 패턴과 일치하는 모든 파일에 규칙이 적용됩니다.

참고: Antigravity는 이제 기본적으로 `.agents/rules`를 사용하지만, `.agent/rules`에 대한 하위 호환성도 여전히 유지합니다.

## @ 멘션 (@ Mentions)

규칙 파일 내에서 `@파일명`을 사용하여 다른 파일을 참조할 수 있습니다. 파일명이 상대 경로인 경우 규칙 파일의 위치를 기준으로 해석됩니다. 파일명이 절대 경로인 경우 실제 절대 경로로 확인되며, 그렇지 않은 경우 리포지토리를 기준으로 해석됩니다. 예를 들어, `@/path/to/file.md`는 먼저 `/path/to/file.md`로 확인을 시도하고, 해당 파일이 존재하지 않는 경우 `workspace/path/to/file.md`로 해석됩니다.
