# 에이전트 스킬 (Agent Skills)

스킬은 에이전트의 기능을 확장하기 위한 [개방형 표준](https://agentskills.io/home)입니다. 스킬은 에이전트가 특정 작업을 수행할 때 따를 수 있는 지침이 포함된 `SKILL.md` 파일을 담고 있는 폴더입니다.

## 스킬이란 무엇인가요?

스킬은 에이전트가 수행할 수 있는 작업을 확장하는 재사용 가능한 지식 패키지입니다. 각 스킬은 다음을 포함합니다:

- 특정 유형의 작업에 어떻게 접근해야 하는지에 대한 **지침(Instructions)**  
- 준수해야 할 **모범 사례(Best practices)** 및 컨벤션  
- 에이전트가 사용할 수 있는 **선택적 스크립트 및 리소스**

대화를 시작하면 에이전트는 사용 가능한 스킬 목록과 각 스킬의 이름 및 설명을 확인합니다. 스킬이 사용자의 작업과 관련된 것으로 보이면 에이전트는 전체 지침을 읽고 이를 따릅니다.

## 스킬이 저장되는 위치

Antigravity는 두 가지 유형의 스킬을 지원합니다:

| 위치 | 범위 |
| :---- | :---- |
| `<workspace-root>/.agents/skills/<skill-folder>/` | 워크스페이스 전용 (Workspace-specific) |
| `~/.gemini/config/skills/<skill-folder>/` | 전역 (Global - 모든 워크스페이스) |

**워크스페이스 스킬**은 팀의 배포 프로세스나 테스트 컨벤션과 같은 프로젝트별 특정 워크플로우에 유용합니다.

**전역 스킬**은 모든 프로젝트에서 작동합니다. 어디서나 사용하고 싶은 개인 유틸리티나 범용 도구에 사용하세요.

참고: Antigravity는 이제 `.agents/skills`를 기본값으로 사용하지만, `.agent/skills`에 대한 하위 호환성도 계속 유지합니다. 

## 스킬 생성하기

스킬을 생성하려면 다음 단계를 따르세요:

1. 스킬 디렉터리 중 하나에 스킬 폴더를 생성합니다.  
2. 해당 폴더 내에 `SKILL.md` 파일을 추가합니다.

```
.agents/skills/
└─── my-skill/
    └─── SKILL.md
```

모든 스킬은 상단에 YAML 프론트매터(frontmatter)가 있는 `SKILL.md` 파일이 필요합니다:

```
---
name: my-skill
description: Helps with a specific task. Use when you need to do X or Y.
---

# My Skill

Detailed instructions for the agent go here.

## When to use this skill

- Use this when...
- This is helpful for...

## How to use it

Step-by-step guidance, conventions, and patterns the agent should follow.
```

### 프론트매터 필드

| 필드 | 필수 여부 | 설명 |
| :---- | :---- | :---- |
| `name` | 아님 | 스킬의 고유 식별자 (소문자, 공백은 하이픈으로 대체). 제공되지 않을 경우 폴더 이름이 기본값으로 사용됩니다. |
| `description` | 예 | 스킬이 수행하는 작업과 사용 시점에 대한 명확한 설명입니다. 에이전트가 스킬 적용 여부를 결정할 때 이 설명을 참조합니다. |

팁: 설명은 3인칭 시점으로 작성하고, 에이전트가 스킬의 연관성을 인식하는 데 도움이 되는 키워드를 포함하세요. 예시: "pytest 컨벤션을 사용하여 Python 코드용 단위 테스트를 생성합니다."

## 스킬 폴더 구조

`SKILL.md`가 유일한 필수 파일이지만, 다음과 같은 추가 리소스를 포함할 수 있습니다:

```
.agents/skills/my-skill/
├─── SKILL.md       # 주요 지침 (필수)
├─── scripts/       # 보조 스크립트 (선택 사항)
├─── examples/      # 참조 구현체 (선택 사항)
└─── resources/     # 템플릿 및 기타 자산 (선택 사항)
```

에이전트는 스킬의 지침을 따를 때 이러한 파일들을 읽을 수 있습니다.

## 에이전트가 스킬을 사용하는 방식

스킬은 **점진적 공개(progressive disclosure)** 패턴을 따릅니다:

1. **탐색 (Discovery)**: 대화가 시작되면 에이전트는 사용 가능한 스킬 목록과 이름 및 설명을 확인합니다.  
2. **활성화 (Activation)**: 스킬이 사용자의 작업과 관련이 있다고 판단되면 에이전트는 전체 `SKILL.md` 내용을 읽습니다.  
3. **실행 (Execution)**: 에이전트는 작업을 수행하는 동안 스킬의 지침을 따릅니다.

사용자가 에이전트에게 특정 스킬을 사용하도록 명시적으로 지시할 필요는 없으며, 에이전트가 맥락에 따라 스스로 결정합니다. 다만, 스킬을 반드시 사용하게 하려면 스킬 이름을 명시적으로 언급할 수 있습니다.

## 모범 사례

### 스킬을 한 가지 목표에 집중시키기

각 스킬은 한 가지 일을 잘 수행해야 합니다. "모든 것을 처리하는" 하나의 스킬 대신, 개별 작업에 따라 별도의 스킬을 만드세요.

### 명확한 설명 작성하기

설명은 에이전트가 스킬의 사용 여부를 결정하는 기준이 됩니다. 스킬이 무엇을 하고 언제 유용한지 구체적으로 작성하세요.

### 스크립트를 블랙박스로 사용하기

스킬에 스크립트가 포함된 경우, 전체 소스 코드를 읽기보다는 먼저 `--help`를 실행하여 스크립트를 실행하도록 유도하세요. 이를 통해 에이전트가 작업에만 집중하여 맥락을 유지하도록 돕습니다.

### 의사결정 트리 포함하기

복잡한 스킬의 경우, 상황에 따라 에이전트가 적절한 접근 방식을 선택할 수 있도록 돕는 섹션을 추가하세요.

## 예시: 코드 리뷰 스킬

에이전트의 코드 리뷰를 돕는 간단한 스킬 예시입니다:

```
---
name: code-review
description: Reviews code changes for bugs, style issues, and best practices. Use when reviewing PRs or checking code quality.
---

# 코드 리뷰 스킬

코드를 리뷰할 때 다음 단계를 따르세요:

## 리뷰 체크리스트

1. **올바름 (Correctness)**: 코드가 의도한 대로 동작하나요?
2. **예외 사항 (Edge cases)**: 오류 조건이 처리되었나요?
3. **스타일 (Style)**: 프로젝트 컨벤션을 따르고 있나요?
4. **성능 (Performance)**: 명백하게 비효율적인 부분이 있나요?

## 피드백 제공 방법

- 변경이 필요한 부분에 대해 구체적으로 작성하세요.
- 단순한 내용 전달을 넘어 그 이유를 설명해 주세요.
- 가능한 경우 대안을 제시해 주세요.
```
