# 아티팩트 (Artifacts)

**아티팩트**는 에이전트가 작업을 수행하고 진행 상황과 사고 과정을 사람(사용자)에게 전달하기 위해 생성하는 구조화된 산출물입니다. 아티팩트에는 서식이 있는 마크다운 계획서(구현 계획서), 코드 차이점(diffs), 아키텍처 다이어그램, 이미지, 그리고 브라우저 녹화본이 포함됩니다.

에이전트가 더 높은 자율성을 갖고 장시간 동안 복잡한 작업을 실행함에 따라, 아티팩트는 비동기식 협업을 가능하게 만듭니다. 개별 도구 호출이나 단계들을 실시간으로 밀접하게 모니터링할 필요 없이, 주요 마일스톤(이정표) 단계에서 높은 수준의 산출물을 검토하면 됩니다.

## 다양한 환경에서의 아티팩트 검토

아티팩트는 주로 에이전트의 **계획 모드(Planning Mode)** 중에 생성되며, Antigravity 2.0 데스크톱 앱과 Antigravity CLI 모두에서 접근할 수 있습니다.

### Antigravity 2.0
데스크톱 앱에는 아티팩트를 표시, 구성 및 관리하는 데 최적화된 시각적인 사이드바와 검토 창이 포함되어 있습니다.
*   **기능**: 대화형 계획서를 검사하고, 시각적인 코드 변경점(diff)을 검토하며, 앱 인터페이스 내부에서 에이전트의 UI 작업 브라우저 녹화본을 바로 재생할 수 있습니다.

### Antigravity CLI
가벼운 터미널 인터페이스인 CLI에서는 빠르고 키보드 중심의 검토 패널을 사용하여 아티팩트를 관리합니다.
*   **워크플로우**: 에이전트가 사용자의 승인이 필요한 파일을 생성하거나 수정할 때, 터미널 상태 표시줄에 알림이 나타납니다.

## 대화형 가이드 및 피드백 (Steering and Feedback)

피드백은 아티팩트 워크플로우의 핵심 메커니즘입니다. 설정에 따라 에이전트는 중간 마일스톤 단계에서 실행을 멈추고, 계획이나 코드 수정을 실행하기 전에 사용자의 검토를 요청합니다.

*   **에이전트 가이드하기**: 아티팩트(예: 구현 계획서)가 목표와 일치하지 않는 경우, 에이전트가 로컬 파일을 수정하기 전에 에이전트의 사고 방향을 올바른 방향으로 이끌기 위한 텍스트 피드백을 직접 입력할 수 있습니다.
*   **세밀한 제어**: 이러한 승인 루프는 사용자가 주도권을 유지할 수 있게 해주며, 에이전트가 높은 자율성을 바탕으로 작동하되 사람이 개입하는 철저한 검증(human-in-the-loop validation)을 유지하도록 보장합니다.
