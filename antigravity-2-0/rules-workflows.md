# 규칙

규칙은 에이전트가 준수해야 할 로컬 및 전역 수준의 수동 정의 제약 조건입니다. 규칙을 통해 사용자는 자신의 고유한 사용 사례 및 스타일에 맞는 특정 동작을 따르도록 에이전트를 안내할 수 있습니다.

규칙을 시작하려면 다음 단계를 따르세요:
1. 에디터 에이전트 패널 상단의 "..." 드롭다운 메뉴를 통해 Customizations(사용자 정의) 패널을 엽니다.
2. Rules(규칙) 패널로 이동합니다.
3. 새로운 전역 규칙을 생성하려면 **+ Global**을 클릭하고, 특정 워크스페이스 전용 규칙을 생성하려면 **+ Workspace**를 클릭합니다.

규칙 자체는 단순한 Markdown 파일로, 에이전트가 사용자의 작업, 기술 스택 및 스타일에 맞게 가이드될 수 있도록 제약 조건을 입력할 수 있습니다.

규칙 파일의 크기는 개당 12,000자로 제한됩니다.

## 전역 규칙

전역 규칙은 `~/.gemini/GEMINI.md`에 위치하며 모든 워크스페이스에 적용됩니다.

## 워크스페이스 규칙

워크스페이스 규칙은 워크스페이스 또는 git 루트의 `.agents/rules` 폴더에 저장됩니다.

규칙 수준에서 규칙이 활성화되는 방식을 정의할 수 있습니다:

- 수동 (Manual): 에이전트의 입력창에서 @멘션을 통해 규칙을 수동으로 활성화합니다.
- 항상 켬 (Always On): 규칙이 항상 적용됩니다.
- 모델 결정 (Model Decision): 규칙에 대한 자연어 설명을 바탕으로 모델이 규칙 적용 여부를 결정합니다.
- Glob: 사용자가 정의한 glob 패턴(예: `*.js`, `src/**/*.ts`)을 기준으로 해당 패턴과 일치하는 모든 파일에 규칙이 적용됩니다.

참고: Antigravity는 이제 `.agents/rules`를 기본값으로 사용하지만, `.agent/rules`에 대한 하위 호환성도 계속 유지합니다. 

## @ 멘션 (@ Mentions)

규칙 파일에서 `@파일명` 형식을 사용하여 다른 파일을 참조할 수 있습니다. 파일명이 상대 경로인 경우 규칙 파일의 위치를 기준으로 해석됩니다. 파일명이 절대 경로인 경우 실제 절대 경로로 확인되고, 그렇지 않으면 리포지토리를 기준으로 확인됩니다. 예를 들어, `@/path/to/file.md`는 먼저 `/path/to/file.md`로 확인을 시도하며, 해당 파일이 존재하지 않는 경우 `workspace/path/to/file.md`로 확인됩니다.

# 워크플로우 (Workflows)

워크플로우를 사용하면 서비스 배포나 PR 댓글 응답 등 반복적인 작업 세트를 수행하도록 에이전트를 안내하는 일련의 단계를 정의할 수 있습니다. 이러한 워크플로우는 markdown 파일로 저장되므로 주요 프로세스를 간편하게 반복 실행할 수 있는 방법을 제공합니다. 한 번 저장된 워크플로우는 에이전트 내에서 `/워크플로우-이름` 형식의 슬래시 명령어를 통해 호출할 수 있습니다.

규칙(Rules)이 프롬프트 수준에서 지속적이고 재사용 가능한 맥락을 제공하여 모델에 가이드를 제공한다면, 워크플로우(Workflows)는 궤적(trajectory) 수준에서 단계 또는 프롬프트의 구조화된 시퀀스를 제공하여 상호 연결된 일련의 작업이나 행동을 통해 모델을 안내합니다.

워크플로우를 생성하려면 다음 단계를 따르세요:

1. 에디터 에이전트 패널 상단의 "..." 드롭다운 메뉴를 통해 Customizations(사용자 정의) 패널을 엽니다.
2. Workflows(워크플로우) 패널로 이동합니다.
3. 모든 워크스페이스에서 액세스할 수 있는 새로운 전역 워크플로우를 생성하려면 **+ Global** 버튼을 클릭하고, 현재 워크스페이스에 특화된 워크플로우를 생성하려면 **+ Workspace** 버튼을 클릭합니다.

워크플로우를 실행하려면 에이전트에서 `/워크플로우-이름` 명령어를 사용하여 호출하면 됩니다. 워크플로우 내부에서 다른 워크플로우를 호출할 수도 있습니다! 예를 들어, `/workflow-1`에 “`/workflow-2` 호출” 및 “`/workflow-3` 호출”과 같은 지침을 포함할 수 있습니다. 호출 시, 에이전트는 워크플로우에 정의된 각 단계를 순차적으로 처리하며 지정된 대로 작업을 수행하거나 응답을 생성합니다.

워크플로우는 markdown 파일로 저장되며 제목, 설명 및 에이전트가 따를 구체적인 지침이 포함된 일련의 단계들로 구성됩니다. 워크플로우 파일의 크기는 개당 12,000자로 제한됩니다.

## 에이전트가 생성하는 워크플로우

에이전트에게 워크플로우를 생성해 달라고 요청할 수도 있습니다! 특히 일련의 단계를 거쳐 에이전트와 수동으로 작업한 후에 효과적이며, 에이전트가 대화 기록을 사용하여 워크플로우를 생성할 수 있습니다.
