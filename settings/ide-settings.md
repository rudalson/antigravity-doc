# 설정 (Settings)

Antigravity 에이전트가 사용자 환경과 상호 작용하고, 명령을 실행하며, 워크스페이스를 안전하게 보호하는 방법을 구성합니다.

## 명령 실행 및 파일 액세스

### 터미널 명령 자동 실행 (Terminal Command Auto Execution)

에이전트가 생성된 셸 명령을 실행하는 방법을 제어합니다:
* **검토 요청(Request Review)**: 에이전트는 터미널 명령을 실행하기 전에 항상 확인 프롬프트를 표시합니다(구성 가능한 허용 목록에 명시적으로 추가된 명령은 제외).
* **항상 진행(Always Proceed)**: 에이전트가 프롬프트 없이 자동으로 명령을 실행합니다(구성 가능한 차단 목록에 명시적으로 추가된 명령은 제외). 자율성이 높지만 위험성도 높습니다.

### 에이전트 비워크스페이스 파일 액세스 (Agent Non-Workspace File Access)

에이전트가 활성 프로젝트 폴더 외부의 파일을 보고 편집할 수 있도록 허용합니다.
* 기본적으로 에이전트는 프로젝트 내부 폴더와 애플리케이션의 로컬 앱 데이터 디렉터리인 `~/.gemini/antigravity/`(아티팩트, 지식 항목 등이 포함됨)에만 액세스할 수 있습니다.
* 이 경계를 강제하면 로컬의 민감한 데이터를 보호할 수 있습니다. 비워크스페이스 액세스 허용은 신중하게 활성화하세요.

## 엄격 모드 (Strict Mode)

엄격 모드는 에이전트에 대한 강화된 보안 제어를 제공하여 외부 리소스 및 민감한 작업에 대한 액세스를 제한할 수 있도록 합니다. 엄격 모드가 활성화되면 환경을 보호하기 위해 여러 보안 조치가 강제 적용됩니다.

### 브라우저 URL 허용 목록/차단 목록 (Browser URL Allowlist/Denylist)

엄격 모드에서 에이전트가 외부 웹사이트와 상호 작용하는 기능은 브라우저의 허용 목록 및 차단 목록에 의해 제어됩니다. 이는 다음 항목에 적용됩니다:
- **외부 마크다운 이미지**: 에이전트는 허용된 URL의 이미지만 렌더링합니다.
- **Read URL 도구**: Read URL 도구는 허용된 URL에 대해서만 자동으로 실행됩니다.

### 터미널, 브라우저 및 아티팩트 검토 정책 (Terminal, Browser, and Artifact Review Policies)

엄격 모드는 터미널, 브라우저 및 아티팩트 상호 작용에 대해 다음 동작을 강제합니다:
- **터미널 자동 실행**: "Request Review(검토 요청)"로 설정됩니다. 에이전트는 터미널 명령을 실행하기 전에 항상 권한을 요청합니다. 엄격 모드가 활성화되면 터미널 허용 목록은 무시됩니다.
- **브라우저 Javascript 실행**: "Request Review(검토 요청)"로 설정됩니다. 에이전트는 브라우저에서 자바스크립트를 실행하기 전에 항상 권한을 요청합니다.
- **아티팩트 검토**: "Request Review(검토 요청)"로 설정됩니다. 에이전트는 아티팩트에 배치된 계획에 따라 조치를 취하기 전에 항상 확인을 요청합니다.

### 파일 시스템 액세스 (File System Access)

엄격 모드는 에이전트가 승인된 파일하고만 상호 작용하도록 파일 시스템 액세스를 제한합니다:
- **.gitignore 준수**: 에이전트는 `.gitignore` 규칙을 준수하여 무시된 파일에 액세스하지 못하도록 방지합니다.
- **워크스페이스 격리**: 워크스페이스 외부 파일에 대한 액세스가 비활성화됩니다. 에이전트는 지정된 워크스페이스 내의 파일만 보고 편집할 수 있습니다.

## 터미널 샌드박싱 (Terminal Sandboxing)

샌드박싱은 에이전트가 실행하는 터미널 명령에 대해 커널 수준의 격리를 제공합니다. 활성화되면 명령은 제한된 파일 시스템 및 네트워크 액세스를 갖는 제한된 환경에서 실행되므로 의도하지 않은 수정으로부터 시스템을 보호합니다.

샌드박싱은 현재 기본적으로 비활성화되어 있지만 향후 릴리스에서 변경될 수 있습니다. macOS 및 Linux에서 지원됩니다. macOS에서는 Apple의 커널 수준 샌드박싱 메커니즘인 Seatbelt(`sandbox-exec`)를 활용합니다. Linux에서는 프로세스 격리를 위해 `nsjail`을 사용합니다.

### 샌드박싱 활성화

Antigravity 사용자 설정에서 샌드박싱을 활성화하거나 비활성화할 수 있습니다. "Enable Terminal Sandboxing"을 전환하여 샌드박싱을 켜거나 끌 수 있습니다. 활성화되면 "Sandbox Allow Network" 토글을 사용하여 네트워크 액세스를 별도로 제어할 수도 있습니다.

![샌드박스 설정 토글](/assets/image/docs/sandbox-settings-toggle.png)

### 제한 사항

샌드박싱이 활성화되면 에이전트의 터미널 명령은 다음 제한 사항의 적용을 받습니다:

- **파일 시스템**: 명령은 지정된 워크스페이스 디렉터리와 필수 시스템 위치에만 쓸 수 있습니다. 이를 통해 에이전트가 실수로 프로젝트 외부의 파일을 삭제하거나 수정하는 것을 방지합니다.

  ![샌드박스에 의해 차단된 파일 시스템 작업](/assets/image/docs/sandbox-filesystem-denied.png)

- **네트워크 액세스**: 네트워크 연결을 독립적으로 제어할 수 있습니다. Antigravity 사용자 설정에서 "Sandbox Network Access" 토글을 사용하여 파일 시스템 제한을 유지하면서 네트워크 액세스를 허용하거나 거부할 수 있습니다.

네트워크 제한으로 인해 명령이 차단되는 예시는 다음과 같습니다:

![샌드박스 네트워크 거부 예시](/assets/image/docs/sandbox-network-denied.png)

### 엄격 모드와의 상호 작용

엄격 모드가 활성화되면 네트워크 액세스가 거부된 상태로 샌드박싱이 자동으로 활성화됩니다. 이를 통해 엄격한 환경에서 작동할 때 최대의 보호를 보장합니다.

![엄격 모드에서의 샌드박스 설정](/assets/image/docs/sandbox-secure-mode-settings.png)
