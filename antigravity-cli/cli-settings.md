# 설정, 렌더링 및 단축키

영구적인 환경 설정을 구성하고, 키보드 단축키를 맞춤 설정하고, 터미널 디스플레이 버퍼를 토글하며, 런타임 CLI 매개변수 재정의를 관리하세요.

## 환경 설정 구성

Antigravity CLI는 상위 호환성이 보장되는 가벼운 JSON 구성 프로필에 사용자 선호 설정을 저장합니다.

### 설정 파일 위치

영구 설정은 일반 JSON 형식으로 저장됩니다.

```text
~/.gemini/antigravity-cli/settings.json
```

CLI는 시스템 기본값과 다른 값만 디스크에 기록하는 **희소 영속성(sparse persistence)**을 활용합니다. 이를 통해 구성 파일을 깔끔하고 가볍게 유지할 수 있으며, 향후 업데이트에 대한 완벽한 상위 호환성을 제공합니다.

### 대화형 설정 패널

원본 JSON 파일을 열지 않고 활성화된 터미널 세션 내에서 직접 설정을 편집하려면 다음과 같이 하십시오.

1. 프롬프트 패널에 `/config`(또는 별칭 `/settings`)를 입력하고 `Enter`를 누릅니다.
2. 전체 화면 **설정 에디터 오버레이(Settings Editor Overlay)**가 열립니다.
3. `↑`/`↓`를 사용하여 제공되는 옵션 사이를 탐색합니다.
4. 하이라이트된 매개변수에서 `Enter`를 눌러 상태를 토글하거나 텍스트 입력 필드를 엽니다.
5. `Esc`를 누르면 변경 사항이 저장되고 에디터가 닫힙니다.

## 명령줄 재정의 

CLI 명령 플래그를 사용하여 개별 터미널 세션의 영구 환경 설정을 일시적으로 재정의할 수 있습니다.

```bash
agy --sandbox=false --notifications=false
```

재정의 플래그가 활성화된 경우, 대화형 `/config` 메뉴의 수정된 설정 옆에 다음과 같은 경고 표시가 나타납니다.

```text
! 도구 권한: strict (명령 플래그에 의해 재정의됨)
```

이러한 세션 중에도 디스크의 영구 설정값을 편집할 수 있지만, CLI는 세션이 닫힐 때까지 활성화된 런타임 플래그 재정의를 강제로 적용합니다.

## 시각적 렌더링 모드

TUI는 터미널 기능 및 연결 대기 시간에 따라 두 가지 시각적 렌더링 모드 중 하나로 작동합니다.

### 대체 화면 모드 (`always`)

이 모드는 터미널의 대체 버퍼(alternate buffer)를 사용하는 전용 디스플레이 화면을 열어 몰입감 있는 독립형 앱 인터페이스를 구성합니다.

- **주요 기능**: 통합 스크롤백(scrollback), 마우스 휠 스크롤 지원, 커스텀 렌더링 스크롤바, 종료 시 터미널 상태의 깔끔한 복원.
- **최적의 사용 환경**: 고급 터미널 에뮬레이터(예: iTerm2, Ghostty, WezTerm)에서의 표준 로컬 개발 세션.

### 인라인 모드 (`never`)

이 모드는 터미널의 표준 stdout 파이프라인에 출력을 직접 순차적으로 렌더링합니다.

- **주요 기능**: 에뮬레이터의 고유 스크롤백 버퍼 내에 전체 세션 기록을 그대로 유지하고, 마우스 입력을 캡처하지 않으며, 표준 명령어 출력과 원활하게 함께 작동합니다.
- **최적의 사용 환경**: 원격 SSH 터미널, `tmux`나 `screen` 같은 터미널 멀티플렉서, 대역폭이 낮은 원격 세션.

## 사용자 정의 상태 표시줄 및 터미널 타이틀

고급 TUI 환경 통합을 위해, 실시간 지표를 토글하거나 커스텀 스크립트를 배포하여 동적인 상태 표시줄을 생성하고 터미널 창 타이틀을 수정할 수 있습니다.

- **[상태 표시줄 커스텀 설정](/docs/cli-statusline)**: 상태 표시기 패널을 관리하고 커스텀 포맷의 상태 표시줄 셸 스크립트를 작성하는 방법을 알아봅니다.
- **[터미널 타이틀 커스텀 설정](/docs/cli-title)**: 창 제목 출력을 토글하고 실시간 에이전트 상태를 창 헤더로 전달하는 방법을 알아봅니다.

## 단축키 설정 

단축키를 특정 작업 공간 명령에 매핑하여 TUI의 거의 모든 키보드 단축키를 커스텀 설정할 수 있습니다.

### 단축키 파일 위치

사용자 정의 맵은 메인 설정 프로필과 함께 저장됩니다.

```text
~/.gemini/antigravity-cli/keybindings.json
```

### 형식 및 커스텀 설정

JSON 구조는 단일 TUI 명령 작업을 단축키 시퀀스 배열에 매핑합니다.

```json
{
  "cli.clear_screen": ["ctrl+l"],
  "prompt.insert_newline": ["shift+enter", "ctrl+j"],
  "edit.open_editor": ["ctrl+g"]
}
```

기본 단축키를 완전히 비활성화하려면 해당 작업을 빈 배열 `[]`에 매핑합니다. JSON 스키마 형식이 잘못되었거나 유효하지 않은 경우, CLI는 해당 특정 작업에 대해 시스템 기본값으로 대체하고 유효한 나머지 매핑을 로드합니다.

<Announcement>
icon: warning
iconColor: var(--theme-primary)
color: var(--theme-surface-surface-container)
text: **보호된 키**: `cli.exit`(`Ctrl+D` / `Ctrl+C`) 및 `cli.enter`(`Enter`)와 같은 중요 탐색 단축키는 시스템에 의해 보호되며 비활성화할 수 없습니다.
</Announcement>

### 기본값 복원

모든 키를 시스템 기본값으로 되돌리려면 단축키 프로필을 삭제하기만 하면 됩니다.

```bash
rm ~/.gemini/antigravity-cli/keybindings.json
```
