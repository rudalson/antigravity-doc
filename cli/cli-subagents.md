# 백그라운드 태스크 및 서브에이전트

지연이 발생하는 빌드, 멀티 파일 코드 생성, 리서치 탐색 작업을 백그라운드에서 병렬로 실행되는 에이전트에게 위임하고, 사용자는 활성화된 프로그래밍 흐름을 유지하세요.

## 비동기 실행 모델 

개발자 속도를 극대화하기 위해 Antigravity CLI는 멀티스레드 비동기 실행 아키텍처를 활용합니다. 장시간 실행되는 빌드, 대규모 코드베이스 검색 탐색, 복잡한 여러 파일 편집 등의 작업 중에 터미널 세션을 잠그는 대신, 기본 에이전트는 이러한 작업을 병렬 **서브에이전트(Subagents)** 또는 백그라운드 **태스크(Tasks)**에 위임합니다.

이 위임 모델 덕분에 지연 시간이 긴 AI 프로세스가 완료될 때까지 기다릴 필요가 없습니다. 여러 자율 백그라운드 스레드가 병렬로 검증 태스크를 실행하는 동안 코드를 계속 작성하거나 프롬프트를 제출하거나 파일을 검사할 수 있습니다.

## 에이전트 관리: `/agents` 패널

활성화된 에이전트 계층 구조는 대화형 터미널 인터페이스를 통해 투명하게 시각화되고 관리할 수 있습니다.

### 패널 열기

프롬프트에서 `/agents`를 입력하고 `Enter`를 누르면 대화형 **에이전트 관리자 패널(Agent Manager Panel)**이 열립니다.

### 패널 개요

패널에는 활성, 완료, 강제 종료(killed) 또는 실패한 모든 백그라운드 에이전트의 실시간 체크리스트가 표시됩니다.

- **식별자 (Identifier)**: 대상 서브에이전트의 고유 ID입니다.
- **역할 (Role)**: 에이전트의 전문 역할("Codebase Researcher" 또는 "Database Debugger" 등)입니다.
- **상태 (State)**: 실시간 상태 표시(running, done, killed 또는 error)입니다.
- **단계 (Step)**: 현재 실행 중인 도구 또는 추론 단계에 대한 실시간 요약입니다.

## 심층 모니터링 

특정 백그라운드 에이전트의 내부 추론, 생각 및 로그를 감사하려면 다음과 같이 하십시오.

1. `/agents` 패널을 열고 `↑`/`↓`를 사용하여 대상 에이전트를 하이라이트합니다.
2. `Enter`를 눌러 **서브에이전트 상세 뷰(Subagent Detail View)**를 엽니다.
3. 이 전체 화면 보기에서는 사설 내부 생각(internal thoughts), 도구 호출 및 실행 출력을 포함한 서브에이전트의 전체 추론 로그를 확인할 수 있습니다.
4. `Esc`를 누르면 종료되고 기본 에이전트 관리자 목록으로 돌아갑니다.

## `/tasks`를 사용한 백그라운드 태스크 모니터링

직접 셸 명령어, 테스트 제품군(testing suites) 또는 `/btw`를 통해 시작된 간단한 백그라운드 쿼리와 같은 비에이전트(non-agentic) 백그라운드 작업의 경우, `/tasks` 명령어를 사용합니다.

```text
/tasks
```

태스크 추적 목록에서 다음을 수행할 수 있습니다.

- 표준 비대화형 백그라운드 프로세스를 추적합니다.
- `↑`/`↓`를 사용하여 태스크를 선택하고 `Enter`를 눌러 표준 출력(stdout) 로그를 봅니다.
- 폭주하는 터미널 프로세스를 안전하게 강제 종료합니다.

## 키보드 인체공학 

서브에이전트가 수동 상호작용 또는 도구 권한 승인을 요구할 때 발생하는 컨텍스트 전환 마찰을 줄이기 위해, Antigravity CLI는 고효율 단축키 경로를 제공합니다.

### 상세 "텔레포트" 탐색 (`Ctrl+J`)

서브에이전트가 승인이 필요한 도구(예: 파일 작성 또는 데이터베이스 마이그레이션 실행)를 발견하면 상태 표시줄 알림이 깜박입니다.

- 메인 프롬프트 패널에서 `Alt+J`를 누르면 현재 대화에서 승인 대기 중인 다음 서브에이전트의 상세 뷰로 즉시 "텔레포트"합니다.
- 조치를 승인하거나 거부하고, `Esc`를 누르면 원래의 메인 스레드로 텔레포트해 돌아옵니다.

### "빠른 경로" 확인 (`Ctrl+K`)

활성화된 작업 공간을 벗어나지 않고 에이전트 조치를 즉시 승인하려면 다음과 같이 하십시오.

1. 활성 프롬프트 창 바로 위에 표시된 인라인 상태 알림을 확인합니다. 보류 중인 작업이 요약되어 표시됩니다(예: `Subagent 12 asks to run "npm test"`).
2. 패널을 전환하거나 오버레이를 열지 않고 `Ctrl+K`를 누르면 보류 중인 빠른 경로 작업을 즉시 승인할 수 있습니다.
