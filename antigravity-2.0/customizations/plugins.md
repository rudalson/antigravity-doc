# 플러그인

플러그인은 스킬, 규칙, MCP 서버, 훅(Hook)을 하나의 패키지로 묶어 Antigravity의 기능을 확장할 수 있는 네임스페이스가 지정된 번들입니다.

## 디렉토리 구조

자신만의 플러그인을 만들거나 기존 플러그인을 살펴보려면 다음과 같은 특정 디렉토리 구조를 따릅니다. 플러그인은 `plugin.json` 파일과 다양한 커스텀 유형을 위한 선택적 하위 디렉토리를 포함하는 디렉토리입니다.

```text
plugins/<plugin-name>/
├── plugin.json       # 필수 마커 파일
├── mcp_config.json   # 선택 사항: MCP 서버 정의
├── hooks.json        # 선택 사항: 훅 정의
├── skills/           # 선택 사항: 스킬
│   └── <skill-name>/
│       └── SKILL.md
└── rules/            # 선택 사항: 규칙
    └── <rule-name>.md
```

### 매니페스트 파일 (`plugin.json`)

모든 플러그인은 루트에 `plugin.json` 파일이 있어야 합니다. 이 파일은 해당 디렉토리가 플러그인임을 식별합니다.

```json
{
  "name": "my-custom-plugin"
}
```

`name` 필드는 선택 사항이며, 생략할 경우 디렉토리 이름이 기본값으로 사용됩니다.

## 지원되는 구성 요소

플러그인은 다음과 같은 구성 요소를 포함할 수 있습니다.

1. **스킬**: `skills/` 하위 디렉토리에 위치합니다. 각 스킬은 에이전트를 위한 지침이 포함된 `SKILL.md` 파일을 가져야 합니다.
2. **규칙**: `rules/` 하위 디렉토리에 위치합니다. 에이전트의 행동에 대한 제약 조건이나 지침을 정의하는 마크다운 파일입니다.
3. **MCP 서버**: 플러그인 루트의 `mcp_config.json`을 통해 설정됩니다. 이를 통해 Antigravity를 외부 도구 및 서비스에 연결할 수 있습니다.
4. **훅(Hooks)**: 플러그인 루트의 `hooks.json`을 통해 설정됩니다. 이를 통해 특정 이벤트가 발생할 때 스크립트나 명령을 실행할 수 있습니다.

## 플러그인 추가 방법

Antigravity에 플러그인을 추가하는 방법은 두 가지가 있습니다.

### 1. 번들 플러그인 사용 (Build with Google)

Antigravity에는 Google에서 제작한 다양한 번들 플러그인이 포함되어 있습니다. 사용자 인터페이스에서 직접 이러한 플러그인을 찾아보고 추가할 수 있습니다.

* **커스텀 설정(Customizations)** 페이지로 이동합니다.
* 사용 가능한 Google 제작 플러그인에 대한 자세한 내용은 [Build with Google 페이지](/docs/build-with-google)를 참조하세요.

### 2. 수동으로 플러그인 추가하기

지정된 플러그인 위치 중 하나에 플러그인 폴더를 배치하여 커스텀 플러그인을 수동으로 추가할 수도 있습니다. Antigravity는 이러한 디렉토리를 자동으로 스캔하여 커스텀 설정을 감지하고 로드합니다.

* **작업 공간 수준(Workspace Level)**: 열려 있는 작업 공간의 루트에 있는 `.agents/plugins/` 또는 `_agents/plugins/` 디렉토리 안에 플러그인 폴더를 배치합니다. 이렇게 하면 이 특정 작업 공간에서 작업할 때만 플러그인을 사용할 수 있습니다.
* **전역 수준(Global Level)**: 사용자 홈 디렉토리의 `~/.gemini/config/plugins/` 안에 플러그인 폴더를 배치합니다. 이렇게 하면 모든 작업 공간에서 플러그인이 활성화됩니다.
