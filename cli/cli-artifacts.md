# 아티팩트 검토

로컬 파일 시스템에 변경 사항을 적용하기 전에 생성된 코드를 감사하고, 구현 제안을 검토하고, 줄 단위 피드백 댓글을 달고, 시각적 미디어 자산을 확인합니다.

## 협업 및 공동 지향 

**아티팩트(Artifact)**는 에이전트가 작업을 완수하고 진행 상황과 생각을 사용자에게 알리기 위해 생성하는 구조화된 산출물입니다. 아티팩트에는 상세한 마크다운 개요(예: 구현 계획), 코드 diff, 아키텍처 다이어그램, 시각적 미디어 파일 등이 포함됩니다.

에이전트가 더 오랜 시간 동안 더 높은 자율성을 가지고 작업함에 따라, 아티팩트는 비동기식 협업을 가능하게 만듭니다. 사용자는 에이전트의 모든 개별 도구 실행을 동기식으로 모니터링할 필요가 없습니다. 대신 주요 이정표(milestones)에서 상위 수준의 산출물을 검토합니다.

자율 에이전트는 때때로 경로를 이탈하거나 잘못된 솔루션을 제시(환각 현상)할 수 있으므로, 아티팩트 워크플로는 중요한 대화형 공동 지향(co-steering) 메커니즘 역할을 합니다. 구성에 따라 에이전트는 중간 이정표에서 멈추며, 사용자는 제안된 계획이나 코드 수정을 검토하고, 인라인 댓글을 달고, 변경 사항이 로컬 파일 시스템에 물리적으로 기록되기 전에 에이전트를 다시 올바른 방향으로 안내할 수 있습니다.

TUI는 이러한 자산을 두 개의 대화형 레이어로 나눕니다.

- **아티팩트 선택기 오버레이**: 검토 상태 표시, 빠른 미리보기 토글, 축소 가능한 폴더 등이 포함된 상위 수준의 체크리스트 메뉴입니다.
- **아티팩트 상세 뷰어**: 인라인 댓글 작성, 구문 강조(syntax highlighting), 다이어그램 크기 조정을 지원하는 전체 화면 코드 감사 인터페이스입니다.

## /artifact 개요

에이전트가 파일을 생성하거나 수정하면 TUI 상태 표시줄에 알림(`/artifact to review`)이 업데이트됩니다. 프롬프트 창 안에서 `ctrl+r`을 누르면 전체 화면 **아티팩트 선택기 패널(Artifact Picker Panel)**이 열립니다.

```text
                                                                                                     10 artifacts · /artifact to review
──────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
>
──────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
Action required (10 left)
› □ new release_notes.md   open  approve reject
  utils.py
  □ new performance_report.md
  api_client.py
  config_manager.py
  □ new user_guide.md
  run_tests.py
  data_processor.py
  □ new system_architecture.md
  □ new project_overview.md

Keyboard: ↑/↓ Navigate  y/n Approve/reject  shift+a Approve all  p Preview  esc Done
```

### 상호작용 단축키

다음의 전용 패널 제어 키를 사용하여 파일 체크리스트를 감사할 수 있습니다.

| 키 | TUI 명령어 | 동작 방식 |
| :---------------- | :-------------------- | :--------------------------------------------------------------------------------------------------------------------------------------- |
| **`↑`** / **`↓`** | `nav.scroll_line`     | 목록 항목을 탐색하며 하이라이트된 선택 영역을 위아래로 스크롤합니다. |
| **`h`** / **`l`** | `nav.switch_button`   | 각 행의 인라인 버튼(**open**, **approve**, **reject**) 포커스를 전환합니다 (좌우 방향키도 지원됨). |
| **`p`**           | `confirm.preview`     | **빠른 인라인 파일 미리보기**를 토글합니다. 선택한 행 바로 아래에 12줄로 축소되고 들여쓰기된 코드 블록 미리보기가 열립니다. |
| **`y`**           | `confirm.approve`     | 하이라이트된 파일을 즉시 승인합니다. 상태 표시가 녹색 체크표시(`✓ approved`)로 업데이트됩니다. |
| **`n`**           | `confirm.reject`      | 하이라이트된 파일을 즉시 거절합니다. 상태 표시가 빨간색 가위표(`✗ rejected`)로 업데이트됩니다. |
| **`Shift+A`**     | `confirm.approve_all` | 대기 중인 승인 대상 파일을 한번에 일괄 승인합니다. |
| **`Shift+R`**     | `confirm.reject_all`  | 대기 중인 거절 대상 파일을 한번에 일괄 거절합니다. |
| **`Enter`**       | `nav.confirm`         | 현재 포커스된 활성 버튼을 실행합니다. `open` 버튼이 포커스된 경우, 전체 화면 상세 뷰어를 실행합니다. |
| **`Esc`**         | `nav.escape`          | 현재의 검토 상태를 저장하고, 승인/거절 내역을 에이전트 스레드에 제출한 후 프롬프트 창으로 포커스를 되돌립니다. |

### 코드 파일 vs 시각적 미디어

작업 공간 자산을 구성하기 위해, 선택기는 파일을 형식 유형에 따라 나눕니다.

- **조치 필요 코드 파일**: 명시적인 승인이 필요한 표준 프로그래밍 코드, 설정 파일 및 계획 마크다운 문서입니다.
- **축소 가능한 미디어 서랍**: 시각적 자산 파일(PNG, JPG, WebP, SVG, MP4, WebM 등)은 전용 **"Media"** 서랍 헤더로 그룹화됩니다.
  - **Media** 헤더 행을 하이라이트하고 `Enter`를 누르면 서랍 목록을 확장하거나 축소할 수 있습니다.
  - 특정 미디어 항목을 하이라이트하고 `Enter`를 누르면 운영 체제의 네이티브 미디어 뷰어로 파일을 엽니다.

## 아티팩트 보기

파일의 코드 구조나 제안된 로직을 면밀히 감사하려면, `open`을 선택(또는 하이라이트된 코드 행에서 직접 `Enter` 입력)하여 **아티팩트 상세 뷰어(Artifact Detail Viewer)**를 엽니다.

```text
implementation_plan.md

    2
    3     이 문서는 알파-켄타우리 텔레메트리 수집 파이프라인을 업그레이드하기 위한 매우 상세한 단계별 엔지니어링 구현 계획을 제공합니다.
    4     현재의 격차, 제안된 아키텍처 개선 사항, 실행 일정, 위험 및 검증 절차를 설명합니다.

    6     ## 1. 개요 (Executive Summary)
```

### 감사 및 탐색

- **스크롤**: `j`/`k`(또는 표준 방향키)를 사용하여 페이지 단위 또는 줄 단위로 스크롤할 수 있습니다.
- **경계 점프**: 파일의 맨 위로 가려면 `g`를 누르고, 맨 아래로 가려면 `Shift+G`를 누릅니다.
- **줄 번호 토글**: `l`을 누르면 코드의 줄 번호 표시 영역을 켜거나 꺼서 원본 코드를 보다 깔끔하게 볼 수 있습니다.

### 세부적인 줄 단위 피드백

특정 코드 블록을 수정해야 하는 경우:

1. 커서를 대상 줄에 위치시킵니다.
2. `c`를 누르면 해당 줄에 직접 첨부되는 인라인 여러 줄 텍스트 에디터 버퍼가열립니다.
3. 피드백을 기술한 후 `Esc`를 눌러 저장하고 제출합니다. 해당 줄에 시각적인 댓글 표시(`💬`)가 추가됩니다.
4. 달아둔 피드백을 삭제하려면, 댓글이 있는 줄로 커서를 이동하고 `d`를 누릅니다.

### 커스텀 Mermaid 다이어그램 렌더링

활성 문서에 시스템 흐름도, 데이터베이스 관계 또는 아키텍처 레이아웃이 포함되어 있는 경우:

- **렌더링 모드 순환 (`m`)**: `m`을 누르면 시각적 렌더링 모드가 다음과 같이 순환됩니다.
  - **Kitty Graphics 이미지**: Kitty 호환 터미널 에뮬레이터 내에 다이어그램을 인라인 그래픽으로 직접 렌더링합니다.
  - **ASCII 박스 아트** (기본값): 모든 셸과 호환되는 깔끔하고 가벼운 텍스트 아트로 다이어그램을 렌더링합니다.
  - **Raw 코드**: 원본 마크다운 코드 블록 경계선과 코드를 표시합니다.
- **그래픽 확대/축소**: Kitty 그래픽 이미지 모드가 활성화되어 있을 때 `ctrl+=`를 누르면 이미지를 확대하고, `ctrl+-`를 누르면 축소합니다.

`Esc`를 누르면 상세 뷰어를 닫고 기본 선택기 체크리스트로 돌아갑니다.
