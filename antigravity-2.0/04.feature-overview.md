# Antigravity 2.0 기능

### 프로젝트 (Projects)  

Antigravity 2.0에서 에이전트는 **프로젝트**(기존 Agent Manager에서는 에이전트가 단일 워크스페이스 폴더에만 엄격하게 매핑되었습니다) 단위로 작업합니다.

* **워크트리 지원**: 프로젝트가 Git 워크트리를 기본적으로 지원하므로, 에이전트가 격리된 백그라운드 폴더에서 작업할 수 있습니다.  
* **범위가 지정된 설정 (Scoped Settings)**: 설정 범위가 프로젝트별로 지정되므로 프로젝트마다 다른 보안 설정을 가질 수 있습니다. 즉, 신뢰할 수 있는 프로젝트에는 더 허용적인 설정을 적용하고 신뢰할 수 없는 폴더에는 더 제한적인 보안 설정을 적용할 수 있습니다. 주요 세 가지 사전 설정은 "Default"(기본), "Full machine"(전체 머신), "Unrestricted"(제한 없음)입니다 (전체 목록은 설정 탭을 참조하세요).  
* **범위가 지정된 권한 (Scoped Permissions)**: 프로젝트에 권한 부여를 연결하여 에이전트가 액세스할 수 있는 대상을 제어합니다. 대화 중에 수동으로 부여한 권한을 유지할 수 있으므로, 에이전트가 신뢰할 수 있는 작업을 학습하고 시간이 지남에 따라 더 매끄러운 사용 경험을 제공하게 됩니다.  
* **다중 폴더 액세스**: 하나의 프로젝트가 여러 폴더에서 작업하도록 구성할 수 있으므로, 에이전트가 동일한 대화 내에서 서로 다른 코드베이스를 교차해 작업할 수 있습니다.  

### 프로젝트 외 대화

프로젝트에 구애받지 않고 빠르고 일회성인 대화를 시작해 보세요. 이러한 세션은 격리된 로컬 임시(scratch) 폴더에서 실행됩니다. 자체 설정을 가지며, 전역 권한을 상속하는 것 외에도 자체 권한을 보유합니다.

### 예약된 작업 (Scheduled Tasks) 

사용자가 프로젝트 일정을 미리 계획할 수 있도록 예약된 작업 기능을 도입합니다. 최신 Gemini 3.5 Flash 모델을 활용하여, 사용자는 부재중일 때 에이전트에게 전송할 메시지를 예약할 수 있습니다.  

* **반복 가능**: 시간 기반 트리거를 설정하여 주기적으로 대화를 시작합니다.  
* 작업은 설정한 분 단위로 반복되도록 설정됩니다.

### 기본 설정으로 보호되는 보안

안전하고 강력한 보안 제어 기능을 제공하여 주도권을 사용자에게 부여합니다:

* **대화형 승인**: 기본적으로 에이전트는 터미널 명령을 실행하기 전에 사용자의 명시적인 승인을 요청합니다.  
* **제한된 액세스**: 기본적으로 에이전트는 프로젝트에 제공된 폴더 내에서만 읽고 쓸 수 있습니다. 보안 사전 설정을 “Full Machine”(전체 머신) 또는 “Unrestricted”(제한 없음)로 변경하면 에이전트가 시스템 전체에 대한 읽기 및 쓰기 액세스 권한을 갖게 됩니다.

### 음성 전사 (Voice transcription)

Antigravity에는 실시간 음성 전사 기능이 내장되어 있어, 자연스러운 대화로 에이전트에게 프롬프트를 보내고 피드백을 남길 수 있습니다.

**사용 방법**:

* **시작/중지**: 텍스트 입력창 옆의 마이크 버튼을 클릭하여 녹음을 시작하고, 다시 클릭하여 중지합니다.  
* **실시간 보기**: 말을 하면 입력 필드에 실시간으로 음성이 텍스트로 변환되어 표시됩니다.  
* **단축키**: `Ctrl + M`을 눌러 녹음을 시작할 수 있습니다. 녹음이 끝나면 `Ctrl + M`을 다시 눌러 중지합니다.

**주요 특징**

* **스마트 정리 (Smart Cleanup)**: 일시 정지나 완벽한 문장 표현에 신경 쓰지 않고 자연스럽게 말해 보세요. 녹음이 중지되면 시스템이 전사 결과를 자동으로 정리하여 자체 수정, 반복 및 불필요한 추임새를 유기적인 프롬프트로 다듬어 줍니다.  
* **대화 맥락 인식 (Conversational Awareness)**: 모델이 진행 중인 대화의 맥락을 인식하므로 프로젝트 전용 용어를 사용해도 정확한 결과를 기대할 수 있습니다. 

**사용 가능 범위**  
음성 입력은 모든 주요 상호작용 환경에서 사용할 수 있습니다:

* **에이전트 입력**: 대화를 시작하고 프롬프트 업데이트를 전송할 때.  
* **아티팩트 댓글**: 계획, 코드 diff, 산출물에 대해 정밀한 인라인 피드백을 남길 때.

### JSON 훅 (JSON Hooks)

JSON 훅을 사용하면 Antigravity 에이전트 실행 주기의 중요한 단계에서 사용자 정의 로컬 셸 스크립트를 실행할 수 있습니다. 단순한 JSON 파일을 통해 전역으로 설정하거나 워크스페이스별로 구성하여 도구 호출 전, 모델 응답 후, 또는 루프 정지 조건에서 에이전트의 동작을 가로채고 제어할 수 있습니다.

[JSON 훅 및 규칙 문서 살펴보기](/docs/hooks)

### 브라우저 (Browser)

Antigravity 2.0에서 브라우저 서브에이전트가 개편되었습니다.

* **주문형(On-demand)**: `/browser` 명령을 통해 호출할 수 있습니다.  
* **Chrome DevTools 통합**: 브라우저 서브에이전트는 Chrome DevTools MCP와도 기본적으로 통합됩니다.  
* **비디오 녹화**: 이제 webm 비디오 형태의 녹화를 지원합니다.
