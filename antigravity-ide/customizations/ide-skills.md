# 에이전트 스킬 (Agent Skills)

스킬은 에이전트의 기능을 확장하기 위한 [오픈 표준](https://agentskills.io/home)입니다. 스킬은 에이전트가 특정 작업을 수행할 때 따를 수 있는 지침이 포함된 `SKILL.md` 파일을 포함하는 폴더입니다.

## 스킬이란 무엇인가요?

스킬은 에이전트가 수행할 수 있는 작업을 확장하는 재사용 가능한 지식 패키지입니다. 각 스킬은 다음을 포함합니다:

- 특정 유형의 작업에 접근하는 방법에 대한 **지침(Instructions)**  
- 준수해야 할 **모범 사례(Best practices)** 및 컨벤션  
- 에이전트가 사용할 수 있는 **선택적 스크립트 및 리소스**

대화를 시작하면 에이전트는 사용 가능한 스킬 이름과 설명 목록을 보게 됩니다. 스킬이 사용자의 작업과 관련이 있다고 판단되면 에이전트는 전체 지침을 읽고 이를 따릅니다.

## 스킬 저장 위치

Antigravity는 두 가지 유형의 스킬을 지원합니다:

| 위치 | 범위 |
| :---- | :---- |
| `<workspace-root>/.agents/skills/<skill-folder>/` | 워크스페이스 전용 |
| `~/.gemini/antigravity/skills/<skill-folder>/` | 전역 (모든 워크스페이스) |

**워크스페이스 스킬**은 팀의 배포 프로세스나 테스트 컨벤션과 같이 프로젝트별 워크플로에 매우 적합합니다.

**전역 스킬**은 모든 프로젝트에서 작동합니다. 모든 프로젝트에서 사용하고 싶은 개인용 유틸리티나 범용 도구에 사용하세요.

참고: Antigravity는 이제 기본적으로 `.agents/skills`를 사용하지만, `.agent/skills`에 대한 하위 호환성도 여전히 유지합니다.

## 스킬 만들기

스킬을 만들려면 다음 단계를 따르세요:

1. 스킬 디렉터리 중 하나에 스킬 폴더를 생성합니다.  
2. 해당 폴더 내에 `SKILL.md` 파일을 추가합니다.

```
.agents/skills/
└─── my-skill/
    └─── SKILL.md
```

모든 스킬은 파일 맨 위에 다음과 같은 YAML 프론트매터(Frontmatter)가 포함된 `SKILL.md` 파일이 필요합니다:

```
---
name: my-skill
description: Helps with a specific task. Use when you need to do X or Y.
---

# 나의 스킬 (My Skill)

에이전트를 위한 상세 지침이 여기에 들어갑니다.

## 이 스킬을 사용하는 경우

- 다음 경우에 사용...
- 다음에 유용...

## 사용 방법

에이전트가 따라야 할 단계별 지침, 컨벤션 및 패턴을 정의합니다.
```

### 프론트매터 필드

| 필드 | 필수 여부 | 설명 |
| :---- | :---- | :---- |
| `name` | 필수 아님 | 스킬의 고유 식별자(소문자, 공백은 하이픈으로 대체). 생략할 경우 기본적으로 폴더 이름이 사용됩니다. |
| `description` | 필수 | 스킬이 수행하는 작업과 사용 시기에 대한 명확한 설명입니다. 에이전트가 스킬 적용 여부를 결정할 때 이 필드를 확인합니다. |

팁: 설명을 3인칭 시점으로 작성하고 에이전트가 스킬의 연관성을 인식하는 데 도움이 되는 키워드를 포함하세요. 예: "Generates unit tests for Python code using pytest conventions." (pytest 컨벤션을 사용하여 Python 코드에 대한 단위 테스트를 생성합니다.)

## 스킬 폴더 구조

`SKILL.md`가 유일하게 필수인 파일이지만, 다음과 같이 추가 리소스를 포함할 수 있습니다:

```
.agents/skills/my-skill/
├─── SKILL.md       # 메인 지침 (필수)
├─── scripts/       # 헬퍼 스크립트 (선택 사항)
├─── examples/      # 참조 구현 (선택 사항)
└─── resources/     # 템플릿 및 기타 자산 (선택 사항)
```

에이전트는 스킬의 지침을 따를 때 이러한 파일을 읽을 수 있습니다.

## 에이전트가 스킬을 사용하는 방법

스킬은 **점진적 공개(progressive disclosure)** 패턴을 따릅니다:

1. **검색(Discovery)**: 대화가 시작되면 에이전트는 사용 가능한 스킬 이름과 설명 목록을 확인합니다.  
2. **활성화(Activation)**: 스킬이 작업과 관련이 있는 것으로 보이면 에이전트는 전체 `SKILL.md` 콘텐츠를 읽습니다.  
3. **실행(Execution)**: 에이전트는 작업을 수행하는 동안 스킬의 지침을 따릅니다.

에이전트에게 특정 스킬을 사용하도록 명시적으로 지시할 필요는 없습니다. 에이전트가 컨텍스트를 기반으로 판단하여 결정합니다. 다만, 특정 스킬을 확실히 사용하게 하려면 이름으로 스킬을 언급할 수 있습니다.

## 모범 사례

### 스킬의 초점 유지하기

각 스킬은 한 가지 작업을 잘 수행해야 합니다. "모든 것을 처리하는" 스킬 대신, 개별 작업에 맞는 별도의 스킬을 생성하세요.

### 명확한 설명 작성하기

설명은 에이전트가 스킬 사용 여부를 결정하는 기준이 됩니다. 스킬이 무엇을 하고 언제 유용한지 구체적으로 작성하세요.

### 스크립트를 블랙박스로 사용하기

스킬에 스크립트가 포함되어 있다면, 에이전트가 소스 코드를 전부 읽기보다는 먼저 `--help` 옵션으로 실행해 보도록 권장하세요. 이를 통해 에이전트의 컨텍스트를 작업에 집중된 상태로 유지할 수 있습니다.

### 의사 결정 트리 포함하기

복잡한 스킬의 경우, 에이전트가 상황에 따라 적절한 접근 방식을 선택할 수 있도록 돕는 섹션을 추가하세요.

## 예시: 코드 리뷰 스킬

다음은 에이전트가 코드를 리뷰하는 데 도움을 주는 간단한 스킬 예시입니다:

```
---
name: code-review
description: Reviews code changes for bugs, style issues, and best practices. Use when reviewing PRs or checking code quality.
---

# 코드 리뷰 스킬 (Code Review Skill)

코드를 리뷰할 때는 다음 단계를 따르세요:

## 리뷰 체크리스트

1. **올바름(Correctness)**: 코드가 의도한 대로 작동합니까?
2. **예외 케이스(Edge cases)**: 에러 조건이 처리되었습니까?
3. **스타일(Style)**: 프로젝트 컨벤션을 따르고 있습니까?
4. **성능(Performance)**: 명백한 비효율성이 존재합니까?

## 피드백 제공 방법

- 변경해야 할 사항을 구체적으로 설명하세요.
- 단순히 무엇을 바꾸는지가 아니라 왜 바꿔야 하는지 설명하세요.
- 가능한 경우 대안을 제시하세요.
```
